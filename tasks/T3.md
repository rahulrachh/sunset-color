# T3 — Location Search Component

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS. US-only scope.

**Design principles:**
- Gradient background is primary; everything else is secondary.
- No cards, no chrome, no borders by default. Typography sits directly on the gradient.
- Text ink: `#fff8ef` (warm white) or `#2a1810` (warm near-black). Parent passes the color.

---

## Task: Location Search Component

**Goal:** A React client component with a centered text input that autocompletes US locations as the user types and emits a structured selection.

**Depends on:** T0.

**Files to create:**
- `components/LocationSearch.tsx`
- `app/search-demo/page.tsx` (isolated test harness)

**Interface contract:**
```ts
export type LocationSelection = {
  name: string;       // e.g., "San Francisco"
  state: string;      // e.g., "California"
  lat: number;
  lng: number;
};

export function LocationSearch(props: {
  onSelect: (loc: LocationSelection) => void;
  inkColor?: string;   // text/border color; defaults to "#fff8ef"
}): JSX.Element;
```

**Behavior:**
- Renders a single `<input>` styled like an underline (no box border — just a thin bottom border in `inkColor` at 40% opacity).
- Placeholder: "where are you watching tonight?"
- On keystroke, debounce 250ms then call Open-Meteo geocoding:
  `https://geocoding-api.open-meteo.com/v1/search?name=<query>&count=8&country=US`
- Filter results to `country_code === "US"` (defense-in-depth).
- Render results as a list below the input, each row shows "City, State". Hover and keyboard (↑/↓/Enter/Esc) navigation.
- On selection: call `onSelect`, clear the dropdown (leave the text), and blur.
- No external UI lib — vanilla Tailwind + small useState. Accessible: `role="combobox"`, `aria-autocomplete="list"`, `aria-activedescendant`.
- No request fires for queries shorter than 2 characters.

**Demo page (`app/search-demo/page.tsx`):**
- Renders the component centered on a dark colored background (e.g., a deep-red div).
- Logs `LocationSelection` to the page below the search when chosen.

**Acceptance criteria:**
- Typing "san fr" within ~500ms shows "San Francisco, California" as a result.
- Arrow keys move focus through results visually; Enter selects.
- `app/search-demo/page.tsx` works in isolation.
- No request fires for queries shorter than 2 characters.
- No console warnings or accessibility violations.
