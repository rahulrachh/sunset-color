# T7 — Main Page Assembly

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS.

**Design principles:**
- Gradient is the entire experience. No cards, no borders. Typography sits directly on it.
- Slow transitions (1.5–2s ease-in-out on background changes). Sunsets bloom, they don't snap.
- Honest rendering: grey/overcast days shown as grey with a melancholy descriptor.
- Ink color (`#fff8ef` warm white or `#2a1810` warm black) chosen dynamically per element.
- Typography: Fraunces (serif) for hero, Inter for the rest.

## Task: Main Page Assembly

**Goal:** Wire everything together into the landing experience.

**Depends on:** T1, T2, T3, T4, T5. All must exist and work in isolation before starting this.

**Files to modify / create:**
- `app/page.tsx` (replace the T0 placeholder).
- Optionally `lib/time.ts` for time-of-day helpers.

**Behavior spec:**

**Empty state:**
- Full viewport, soft warm off-white background `#fefcf8`.
- `LocationSearch` centered (flex centering).
- Below the search, in `#999` Inter, very small: "where do you want to watch tonight's sunset?"
- Use ink `#2a1810` (dark) for search in the empty state.

**On `onSelect` from search:**
1. Call `GET /api/forecast?lat=...&lng=...` (date inferred per "Time-of-day" below).
2. Pass result through `forecastToColor`.
3. Animate gradient by setting CSS variables `--g-top`, `--g-mid`, `--g-bottom` on `<main>` (1.5s ease-in-out — easiest with a CSS `transition: background 1500ms ease-in-out` on a `linear-gradient(...)` driven by the vars).
4. Search bar shrinks and moves to top-left over 600ms (becomes a pill: rounded, smaller font, semi-transparent backdrop).
5. Render hero text block, centered:
   - Location name: Fraunces, weight 300, ~64px desktop / ~40px mobile.
   - Sunset time below: Inter, weight 400, ~22px. Format: "8:34 PM tonight" or "8:34 PM tomorrow".
   - Descriptor: Inter italic, weight 400, ~16px, opacity 0.8. Format: "— {descriptor} —".
   - All three centered. Ink: `pickInk(stops, 0.5)`.
6. Render `<CloudLayer>` with forecast cloud values + wind. `pointer-events-none`.
7. On a *second* selection: smoothly transition gradient + text. Do **not** unmount and remount.

**Time-of-day logic (e.g., in `lib/time.ts`):**
- Compute "now" in the location's local timezone (use the `timezone` field from Open-Meteo's response, or `Intl.DateTimeFormat` with the IANA tz).
- If `now < sunset_today - 30min`, show today's sunset. Label "tonight".
- Else, show tomorrow's sunset. Label "tomorrow".

**Acceptance criteria:**
- Fresh load → centered search + whisper text on warm off-white. No clouds, no gradient.
- Select "San Francisco" → bg gradient fades in over ~1.5s. Search pill moves to top-left. Hero text renders centered. Cloud wisps drift if forecast has clouds.
- Select a second city → smooth crossfade to new gradient. No flash, no remount.
- Resize → layout stays centered. No horizontal scroll. Mobile (375px) usable.
- Lighthouse performance > 90 on mobile.
