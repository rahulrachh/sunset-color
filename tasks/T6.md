# T6 — Share Image Endpoint

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, deployed on Vercel.

**Share image strategy:** uses `@vercel/og` to render a 1080×1080 PNG server-side. This works for social-media link previews (og:image) and for a "save tonight's sunset" download button.

---

## Task: Share Image Endpoint

**Goal:** A server endpoint that generates a 1080×1080 PNG of the sunset prediction for a given location.

**Depends on:** T1 (forecast service), T2 (color map). Reuses their functions directly via server-side imports.

**Files to create:**
- `app/api/share/route.tsx` (uses `@vercel/og`, JSX-friendly route file).

**Interface contract:**
```
GET /api/share?lat=37.77&lng=-122.42&date=2026-06-30
→ 200 image/png (1080×1080)
→ 400 invalid params
```

**Image content:**
- Full-bleed 3-stop vertical gradient using `ColorResult.gradient` from T2.
- Centered text block:
  - Location name (large, Fraunces serif — use the `@vercel/og` font loading API to embed Fraunces from Google Fonts).
  - Sunset time (medium, Inter).
  - Descriptor word (small, italic).
  - Small footer at the very bottom: production URL (e.g., "sunset-color.vercel.app").
- Ink color picked via `pickInk` (from T5) at the text's vertical position.

**Implementation notes:**
- `@vercel/og` doesn't render arbitrary cloud SVG cleanly at this stage — skip clouds in the share image. Gradient + text only.
- The route also needs the location's display name — you can get it from a reverse-geocoding call to Open-Meteo or pass it as a `?name=` query param. Prefer the query param to keep the endpoint cheap.
  - Updated contract: `GET /api/share?lat=...&lng=...&name=San+Francisco&date=...`
- Cache header: `Cache-Control: public, immutable, max-age=3600`.
- Validate lat/lng to US bounds (same as T1's API route).

**Acceptance criteria:**
- Visiting `/api/share?lat=37.77&lng=-122.42&name=San+Francisco` in the browser displays a 1080×1080 PNG with a gradient and "San Francisco" rendered.
- Different locations/dates produce visibly different PNGs.
- AirNow key is not exposed in any HTTP response.
