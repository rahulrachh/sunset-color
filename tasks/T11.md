# T11 — First-Visit Guided Tour

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, deployed on Vercel (auto-deploys from GitHub `master`).

**Design principles:**
- Gradient background is primary; everything else is secondary.
- No cards, no chrome, no borders by default. Typography sits directly on the gradient.
- Slow transitions (600ms–2s ease-in-out). Nothing snaps.
- Text contrast computed dynamically via `lib/contrast.ts` (`pickInk`); ink is `#fff8ef` (warm white) or `#2a1810` (warm near-black).
- Typography: Fraunces (serif) for hero, Inter for everything else (already loaded via `next/font` as CSS vars `--font-serif` / `--font-sans`).

**Relevant existing UI (all rendered in `app/page.tsx` once a location is selected):**
- `components/MultiDayStrip.tsx` — bottom-center strip: a **Today** button, a **‹ Back** button, **three day swatches** in the middle (colored circles previewing each day's sunset), and a **Next ›** button.
- The **share button** — circular icon button, bottom-right corner of the page.

**Verification:** use the project skill `.claude/skills/verify/SKILL.md` (dev server + Playwright). Do NOT write test files for this task.

## Task: First-Visit Guided Tour

**Goal:** The first time a visitor selects a location (i.e. the first time the strip and share button actually exist on screen), walk them through the controls with a step-by-step spotlight tour. On any later visit, never show it again.

**Depends on:** T7, T8, T10.

**Tour steps, in order (one popup per step, advanced by a "Next" button on the popup):**

1. **The three day swatches (middle of the strip)** — highlight all three as a group. Copy along the lines of: "Each circle previews a day's sunset — tap one to see that evening."
2. **The Next › button** — "Peek further into the future."
3. **The ‹ Back button** — "Step back a day."
4. **The Today button** — "Jump back to tonight."
5. **The share button (bottom-right)** — "Download tonight's sunset as an image, or share it." The final popup's button reads "Done" instead of "Next".

**Tasks:**

1. **Build `components/Tour.tsx`.**
   - Renders a dim overlay (`rgba(0,0,0,0.45)` or similar) across the page with a **spotlight cutout** around the current step's target element (rounded rect, ~8px padding around the target's bounding box). The overlay must not block the gradient's beauty more than needed — keep it subtle.
   - A small popup/tooltip anchored near the highlighted element (above the strip elements, to the left of the share button) containing: the step text, a step indicator ("2 / 5"), a **Next / Done** button, and a **Skip tour** text button.
   - **Skip tour** ends the tour immediately from any step.
   - Target elements are found via `data-tour` attributes (see task 2). Recompute the spotlight position on window resize.
   - Popup styling follows the design principles: `--font-sans`, small, warm-white background (like the search pill: `rgba(255,248,239,0.9)`), dark ink `#2a1810`, no harsh borders, 300–600ms ease transitions when moving between steps.
   - Accessibility: popup has `role="dialog"` and `aria-label`; Escape key skips the tour; Next/Skip are real `<button>`s and keyboard-focusable; focus moves to the popup when the tour starts and returns to the page when it ends.

2. **Tag the targets.**
   - In `components/MultiDayStrip.tsx`: add `data-tour="days"` to the container wrapping the three day swatches, `data-tour="next"` to the Next button, `data-tour="back"` to the Back button, `data-tour="today"` to the Today button.
   - In `app/page.tsx`: add `data-tour="share"` to the share button.
   - Keep these attributes purely declarative — no behavior changes to the tagged components.

3. **First-visit persistence — use `localStorage`, not cookies.**
   - Key: `sunset-tour-done`, value `"1"`, set when the tour finishes (Done) **or** is skipped.
   - Rationale (record this in a code comment): the flag is only ever read client-side after a user interaction, so `localStorage` is the right tool — no server needs it, no cookie consent implications, nothing sent on every request. Cookies would only be needed if the server had to know.
   - Guard all access with a `typeof window !== "undefined"` check / try-catch (Safari private mode can throw).

4. **Wire it up in `app/page.tsx`.**
   - Trigger: the first time a view is set (location selected) **and** `localStorage.getItem("sunset-tour-done")` is null.
   - Start the tour ~1.5s after the view renders, and only once the strip has painted its swatch placeholders, so the gradient transition finishes first and targets exist. (The day swatches may still be loading their colors — that's fine, the dashed placeholders occupy the same positions.)
   - The tour never triggers again in the same session after finishing/skipping (state), nor in later sessions (localStorage).

**Acceptance criteria:**
- Fresh browser profile (or cleared localStorage): select a city → after the gradient settles, the tour starts at step 1 with the three middle swatches highlighted; clicking Next advances through Next › → ‹ Back → Today → share button; the last popup says Done; clicking Done ends the tour and sets `sunset-tour-done` in localStorage.
- "Skip tour" is visible on every step and ends the tour immediately, also setting the flag.
- Reload the page, select a city again → no tour.
- Escape key skips the tour. Tab reaches Next and Skip.
- Resizing the window while the tour is open keeps the spotlight on its target.
- No console errors. `npm run build` passes.
- Verify all of the above with the `verify` skill (Playwright, `localStorage.clear()` to simulate first visit). Do not add test files.
