# T0 — Project Scaffolding

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor ("fiery peach", "muted", "overcast", etc.) sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, deployed on Vercel. US-only scope.

**Data sources (all free):**
- **Open-Meteo Forecast** — no key. `https://api.open-meteo.com/v1/forecast`. Cloud cover (high/mid/low), humidity, visibility, wind, pressure.
- **Open-Meteo Geocoding** — no key. `https://geocoding-api.open-meteo.com/v1/search`. Filter results to `country_code === "US"`.
- **AirNow** — requires `AIRNOW_API_KEY` env var, server-side only. `https://www.airnowapi.org/aq/forecast/latLong/`. PM2.5 / aerosols.
- **Sunset time** — use Open-Meteo's `daily=sunset` field, or the `suncalc` npm package.

**Design principles:**
- Gradient background is primary; everything else is secondary.
- No cards, no chrome, no borders by default. Typography sits directly on the gradient.
- Slow transitions (1.5–2s ease-in-out on background changes).
- Honest rendering: grey/overcast days shown as grey with a melancholy descriptor, not faked.
- Text contrast computed dynamically; ink is `#fff8ef` (warm white) or `#2a1810` (warm near-black), picked by WCAG luminance.
- Typography: Fraunces (serif) for hero, Inter for everything else. Both from Google Fonts via `next/font`.

## Task: Project Scaffolding

**Goal:** Initialize a working Next.js project at the current directory with the agreed stack, env file, and a successful Vercel deploy. After this task, `npm run dev` shows a placeholder page locally and the same is reachable at a `*.vercel.app` URL.

**Depends on:** nothing.

**Steps:**
1. From the project root, run `npx create-next-app@latest .` with: TypeScript yes, ESLint yes, Tailwind yes, `src/` directory no, App Router yes, Turbopack yes, customize import alias no.
2. Install runtime deps: `npm i suncalc @vercel/og`. Install dev types: `npm i -D @types/suncalc`.
3. Create `.env.local.example` at the root with:
   ```
   AIRNOW_API_KEY=
   ```
   Create `.env.local` with the same line (already gitignored by `create-next-app`).
4. Replace the default `app/page.tsx` with a single centered `<h1>sunset-color</h1>` placeholder.
5. Update `README.md` with: one-sentence project description, local setup steps (`npm i`, copy env example, paste AirNow key, `npm run dev`), and a link to `plan.md`.
6. `git init`, `git add .`, initial commit `chore: scaffold`. Create a GitHub repo (the user will do this manually if not yet created) and push.
7. Connect the repo to Vercel via the Vercel dashboard. Add `AIRNOW_API_KEY` as an env var in Vercel for all environments. Trigger the first deploy.

**Acceptance criteria:**
- `npm run dev` serves a page at `http://localhost:3000` showing "sunset-color".
- Same page live at the Vercel-assigned URL.
- `.env.local` exists locally and is gitignored.
- Vercel project has `AIRNOW_API_KEY` set.

**Hand-off:** record the Vercel URL and the GitHub repo URL in `README.md`.
