# T14 - Vercel Web Analytics

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, deployed on Vercel (auto-deploys from GitHub `master`).

**Design principles:**
- Gradient background is primary; everything else is secondary.
- No cards, no chrome, no borders by default. Typography sits directly on the gradient.
- Slow transitions (600ms-2s ease-in-out). Nothing snaps.
- Text contrast computed dynamically via `lib/contrast.ts` (`pickInk`); ink is `#fff8ef` (warm white) or `#2a1810` (warm near-black).
- Typography: Fraunces (serif) for hero, Inter for everything else (already loaded via `next/font` as CSS vars `--font-serif` / `--font-sans`).

**Relevant existing UI:**
- `app/layout.tsx` - the root layout (server component): fonts, metadata, `<html>`/`<body>` shell.
- `app/page.tsx` - the one page; auto-locates from IP on load (T13), otherwise a centered search.

**Verification:** use the project skill `.claude/skills/verify/SKILL.md` (dev server + Playwright). Do NOT write test files for this task.

## Task: Vercel Web Analytics

**Goal:** Track visitors and page views on the deployed site with Vercel Web Analytics, per the quickstart: https://vercel.com/docs/analytics/quickstart. Privacy-friendly, no cookies, nothing to configure beyond mounting the component.

**Approach - the official `@vercel/analytics` package, Next.js App Router flavor.** The `Analytics` component injects the tracking script; on Vercel the script is served from the app's own origin (`/_vercel/insights/*`), so there is no third-party domain to worry about. The component is a no-op locally except for debug console logs.

**Tasks:**

1. **Install the package.**
   - `npm install @vercel/analytics`.

2. **Mount the component in `app/layout.tsx`.**
   - `import { Analytics } from "@vercel/analytics/next";` (the Next-specific entry point, which adds App Router route support - not `/react`).
   - Render `<Analytics />` inside `<body>`, after `{children}`. No props needed.
   - Touch nothing else in the layout - fonts, metadata, and class names stay as they are.

3. **Dashboard enablement is a user step - do not attempt it.**
   - Web Analytics must be enabled once per project in the Vercel dashboard (project -> Analytics -> Enable). The agent cannot and should not do this; remind the user in the final report. Until it is enabled AND a new deploy ships, the deployed site serves no `/_vercel/insights/script.js` and collects nothing.

**Local behavior to expect (and accept):**
- In dev the component runs in debug mode: it logs `[Vercel Web Analytics]` messages via `console.log` and sends nothing. Debug logs are fine; console *errors* are not.
- `/_vercel/insights/*` routes exist only on Vercel deployments, never on the local dev server.

**Acceptance criteria:**
- `npm install` succeeded; `@vercel/analytics` appears in `package.json` dependencies (lockfile updated).
- The page still renders and behaves at `/`: centered search when locate returns null, auto-locate flow when mocked (spot-check per T13), search flow works (`seat` -> Seattle). Playwright, mocked APIs, as in the verify skill.
- No console errors on load or through a search flow. `[Vercel Web Analytics]` debug *logs* in dev are expected and OK.
- The rendered HTML/DOM at `/` shows evidence the Analytics component mounted (in dev it renders a script tag pointing at the debug script `va.vercel-scripts.com/v1/script.debug.js` - assert on that, or on the debug console log).
- `npx tsc --noEmit` clean. `npm run build` passes (never while a dev server is running - build in an isolated junctioned copy; the user keeps a dev server on port 3000, do not kill it).
- Final report reminds the user of the one manual step: enable Web Analytics on the Vercel project (Analytics tab -> Enable), then verify post-deploy that visiting the prod site fires a request to `/_vercel/insights/view` in the Network tab.
- Do not add test files.
