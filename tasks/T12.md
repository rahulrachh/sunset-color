## Task: Search Autosuggestions

**Goal:** Partial queries must suggest US places. Today, typing `vir` shows "no places found" even though Virginia Beach, Viroqua, Virginia (IL), etc. exist.

**Root cause (verified 2026-07-07 against the live API):**
- The geocoding API **ignores the `country=US` parameter** — it is not a supported filter. `name=vir&count=8` returns eight non-US places (villages named "Vir" in Croatia, Slovenia, Czechia, ...), so the client-side US filter empties the list.
- `name=vir&count=100` returns 5 US matches, including Virginia Beach — the data is there, the request just asks for too few results to survive filtering.
- API behavior for reference: 1-character queries return nothing, 2-character queries only exact-match, 3+ characters do fuzzy/prefix matching. `count` maxes out at 100.

**Tasks:**

1. **Fetch enough results for the US filter to bite.** In `components/LocationSearch.tsx`, request `count=100` and drop the useless `country=US` param. Keep the existing client-side `country_code === "US"` filter as the single source of truth.
2. **Cap what's shown.** Display at most 8 suggestions after filtering, preserving the API's relevance order (it ranks by match quality and population).
3. **Dedupe.** Two results with the same `name` + `admin1` read as duplicates in the dropdown — keep only the first.
4. **Keep the interaction as-is.** Debounce, min query length 2, keyboard navigation, commit behavior, the committed-place placeholder flow, and the "no places found" line must all work unchanged. No visual redesign — same transparent dropdown.

**Acceptance criteria:**
- Typing `vir` shows US suggestions (Virginia Beach first among them), not "no places found".
- Typing `seat` still suggests Seattle; selecting it still loads the forecast view.
- No more than 8 suggestions render at once, with no name+state duplicates.
- Genuinely unmatchable input (e.g. `zzzxqy`) still shows the "no places found" line.
- No console errors. `npm run build` passes (do not run the build while a dev server is running — stop it first).
- Verify with the `verify` skill (Playwright; use the real geocoding API for the `vir` case since the bug lives in the request shape). Do not add test files.
