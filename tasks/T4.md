# T4 — Cloud Rendering Component

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS.

**Design principles:**
- Gradient background is primary; clouds are layered on top.
- No animation libraries — pure SVG + CSS animations.
- Subtle: clouds should feel alive but never demand attention away from the gradient.

---

## Task: Cloud Rendering Component

**Goal:** A purely visual component that renders drifting cirrus, altocumulus, and low haze layers over the page based on forecast cloud cover.

**Depends on:** T0.

**Files to create:**
- `components/CloudLayer.tsx`
- `app/cloud-demo/page.tsx` (test harness with sliders)

**Interface contract:**
```ts
export function CloudLayer(props: {
  highCloudPct: number;   // 0..100
  midCloudPct: number;    // 0..100
  lowCloudPct: number;    // 0..100
  windKph: number;        // wind speed at altitude — affects drift
}): JSX.Element;
```

**Visual spec:**
- Full-screen, pointer-events-none overlay (`absolute inset-0`).
- **High clouds (cirrus)** in top third (`top: 0` to `top: 33%`):
  - Thin, elongated SVG paths with high blur, opacity 0.4–0.7.
  - Number of wisps = `floor(highCloudPct / 15)` (100% → ~6 wisps, 0% → 0).
  - Color: warm off-white (`#fff8ef` at lowered opacity).
  - Drift: horizontal across screen. Period = `60s * (10 / max(windKph, 1))` capped [20s, 120s].
- **Mid clouds (altocumulus)** at `top: 30%` to `top: 60%`:
  - Softer, puffier ellipse-blob shapes. Number = `floor(midCloudPct / 25)`.
  - Opacity 0.5–0.8.
  - Drift at half cirrus speed.
- **Low clouds (haze band)** at `top: 70%` to `top: 100%`:
  - Single broad horizontal band with vertical fade.
  - Opacity scales linearly with `lowCloudPct / 100`, max 0.45.
  - `mix-blend-mode: multiply` to slightly dim the gradient below.
  - No drift.
- Each layer is keyed/seeded by props/indices (not random) so changing `highCloudPct` from 30 → 45 *adds* a wisp instead of reshuffling.

**Demo page (`app/cloud-demo/page.tsx`):**
- Three sliders (high/mid/low cloud %) + one wind slider.
- A colored background div (use a hardcoded deep-orange gradient) so clouds are visible.
- Sliders update the cloud layer in real time.

**Acceptance criteria:**
- At 0% across the board, the component renders nothing visible.
- At 100% high cloud, ~6 wisps drift across the top third.
- No layout shift when props change (everything `position: absolute`).
- No console warnings about hydration or keys.
- Sliders in the demo page update visuals smoothly.
