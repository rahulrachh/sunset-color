# T1 — Forecast Service + API Route

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor ("fiery peach", "muted", "overcast", etc.) sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, deployed on Vercel. US-only scope.

**Data sources (all free):**
- **Open-Meteo Forecast** — no key. `https://api.open-meteo.com/v1/forecast`. Cloud cover (high/mid/low), humidity, visibility, wind, pressure.
- **Open-Meteo Geocoding** — no key. `https://geocoding-api.open-meteo.com/v1/search`. Filter results to `country_code === "US"`.
- **AirNow** — requires `AIRNOW_API_KEY` env var, server-side only. `https://www.airnowapi.org/aq/forecast/latLong/`. PM2.5 / aerosols.
- **Sunset time** — use Open-Meteo's `daily=sunset` field, or the `suncalc` npm package.

**AirNow key location:** `.env.local` at project root, `AIRNOW_API_KEY=...`. Access via `process.env.AIRNOW_API_KEY` server-side only.

## Task: Forecast Service + API Route

**Goal:** A server-only function that, given a lat/lng and an optional date, returns a normalized forecast object combining Open-Meteo and AirNow data. Plus a Next.js API route that exposes it to the client without leaking the AirNow key.

**Depends on:** T0 (scaffolded Next.js project).

**Files to create:**
- `lib/forecast.ts`
- `app/api/forecast/route.ts`
- `lib/forecast.test.ts`

**Interface contract:**
```ts
// lib/forecast.ts
export type ForecastResult = {
  sunset_iso: string;            // ISO 8601, local time of sunset at the location
  high_cloud_pct: number;        // 0-100
  mid_cloud_pct: number;         // 0-100
  low_cloud_pct: number;         // 0-100
  humidity_pct: number;          // 0-100, at 2m AGL, at sunset time
  visibility_km: number;         // at sunset time
  pm25_ugm3: number | null;      // null if AirNow returned no data
  wind_high_kph: number;         // wind at ~500hPa (proxy for high-cloud drift)
  pressure_hpa: number;          // sea-level pressure at sunset time
};

export async function getSunsetForecast(
  lat: number,
  lng: number,
  date?: string  // YYYY-MM-DD, defaults to today (in the location's local tz)
): Promise<ForecastResult>;
```

**Implementation notes:**
- Open-Meteo call should request hourly fields: `cloud_cover_high`, `cloud_cover_mid`, `cloud_cover_low`, `relative_humidity_2m`, `visibility`, `pressure_msl`, `wind_speed_500hPa`. Plus `daily=sunset` and `timezone=auto`.
- After fetching, find the hourly index closest to `sunset_iso` and read those values. Sunset rarely lands exactly on the hour; use the closest hour, or interpolate between the two surrounding hours.
- Visibility from Open-Meteo is in meters; convert to km.
- AirNow returns an array of pollutants; pick the entry with `ParameterName === "PM2.5"` and use `AQI` or `Value`. If AirNow returns `[]` or fails, set `pm25_ugm3: null`. Do not throw.
- All external fetches must have a 5s timeout and a single retry. Use `AbortController`.

**API route (`app/api/forecast/route.ts`):**
```
GET /api/forecast?lat=37.77&lng=-122.42&date=2026-06-30
→ 200 { ...ForecastResult }
→ 400 { error: "missing lat/lng" }
→ 502 { error: "upstream failure" }
```
- Validate `lat` is in [24, 50] and `lng` is in [-125, -66] (continental US bounds). Reject otherwise with 400.
- Add `Cache-Control: public, max-age=900, s-maxage=900` (15min).

**Acceptance criteria:**
- `curl 'http://localhost:3000/api/forecast?lat=37.77&lng=-122.42'` returns valid ForecastResult JSON.
- AirNow key is never present in the browser's network response.
- Bad coords return 400.
- AirNow being down does not break the response; `pm25_ugm3` is `null`.
- `lib/forecast.test.ts` includes a mocked-fetch unit test for the "find closest hour" logic. `npm test` passes (configure Vitest or Jest if not already set up).
