# T5 — Contrast Utility + Typography

## Shared Context

You are working on **sunset-color**, a one-page website that predicts the color of the sky at sunset for a US location and renders the prediction as the page background.

**Concept:** the forecast IS the design. A 3-stop vertical gradient shows predicted sunset colors. Drifting wispy SVG shapes show forecast cloud cover at high/mid/low altitudes. A small italic descriptor sits below the location name and sunset time.

**Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS.

**Design principles:**
- Typography sits directly on a gradient — no card backing.
- Ink color is dynamically chosen for contrast: `#fff8ef` (warm white) or `#2a1810` (warm near-black). Never pure white/black.
- Typography: Fraunces (serif) for hero, Inter (sans) for everything else.

## Task: Contrast Utility + Typography

**Goal:** Pick text ink color based on the gradient behind it, and load Fraunces + Inter site-wide.

**Depends on:** T0. (Will be consumed by T2 for `GradientStop` type, but implement defensively — if T2 doesn't exist yet, inline the type.)

**Files to create / modify:**
- `lib/contrast.ts` (new)
- `lib/contrast.test.ts` (new)
- `app/layout.tsx` (modify — add fonts)
- `tailwind.config.ts` (modify — register font CSS vars)
- `app/contrast-demo/page.tsx` (new, test harness)

**Interface contract:**
```ts
// If lib/colorMap.ts (T2) exists, import GradientStop from there.
// Otherwise inline this type:
type GradientStop = { color: string; position: number };

export const INK_LIGHT = '#fff8ef';
export const INK_DARK  = '#2a1810';

// position: 0 (top) to 1 (bottom). Samples gradient at that y-position.
export function pickInk(stops: GradientStop[], position: number): typeof INK_LIGHT | typeof INK_DARK;

// Direct version for a single bg hex.
export function pickInkFor(bgHex: string): typeof INK_LIGHT | typeof INK_DARK;
```

**Implementation:**
- `pickInkFor`: convert hex → relative luminance (WCAG 2.1 formula). If luminance > 0.45 → return `INK_DARK`, else `INK_LIGHT`.
- `pickInk`: lerp between the two adjacent stops surrounding `position`, then call `pickInkFor`.
- Tests cover: midpoint between two stops, exact stop position, lerp correctness for known hex pairs, luminance threshold behavior at boundaries.

**Typography in `app/layout.tsx`:**
- Use `next/font/google` to load **Fraunces** (variable, weights 300–700) and **Inter** (variable, weights 400–600).
- Expose as CSS variables `--font-serif` and `--font-sans`.
- In `tailwind.config.ts`, extend `theme.fontFamily` with `serif: ['var(--font-serif)']` and `sans: ['var(--font-sans)']`.
- Default body uses `font-sans`.

**Demo page (`app/contrast-demo/page.tsx`):**
- ~8 colored swatches (light yellows, deep reds, slate greys, mid-tones) each containing sample text using the auto-picked ink. All text should remain legible.

**Acceptance criteria:**
- Tests pass.
- `app/contrast-demo/page.tsx` shows swatches with auto-picked ink, all legible.
- Visiting any page with serif text renders Fraunces (verify visually).
- No layout shift from font loading (use `next/font` `display: 'swap'` defaults).
